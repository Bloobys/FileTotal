# 🛡️ FileTotal

מערכת **FileTotal** הוא סורק קבצים המבוסס על ארכיטקטורת **Client-Server**. המערכת מבצעת ניתוח רב-שכבתי לקבצים חשודים ומנגישה למשתמש את רמת הסיכון בשפה פשוטה וברורה באמצעות בינה מלאכותית (Google Gemini).

---

## 🔍 מה המערכת עושה?
המערכת משלבת מספר טכנולוגיות כדי לתת תשובה חד-משמעית לגבי בטיחות הקובץ:
* **VirusTotal API:** בדיקה מול עשרות מנועי אנטי-וירוס מובילים.
* **אימות Magic Number:** זיהוי זיופי סיומות (למשל קובץ EXE שמתחזה ל-PDF).
* **ניתוח סטטי (PE & Office):** סריקת המבנה הפנימי של הקובץ וחיפוש פקודות מאקרו או פונקציות חשודות.
* **סיכום AI (Google Gemini):** איסוף כלל הנתונים הטכניים והפיכתם להמלצה פשוטה למשתמש.

---

## 📁 מבנה הפרויקט ותפקידי הקבצים

| קובץ | תפקיד במערכת |
| :--- | :--- |
| `SERVER.py` | השרת המרכזי המקבל את הקבצים ומפעיל את מנועי הסריקה. |
| `CLIENT.py` | ממשק המשתמש (GUI) המאפשר העלאת קבצים וקבלת דוח סיכום. |
| `checkVT.py` | מודול האחראי על השאילתות מול VirusTotal. |
| `checkMNandSort.py` | אימות סוג הקובץ ברמת הבינארי. |
| `checkPEfiles.py` | ניתוח מעמיק של קבצי הרצה (EXE/DLL). |
| `checkDUCC.py` | סורק קבצי PDF ומסמכי Office. |
| `Caesar.py` | מחלקה להצפנת ופענוח נתונים (Caesar Cipher). |
| `CREATE_DB.py` | סקריפט ליצירת מסד הנתונים הראשוני. |

## 🛠️ דרישות מערכת והתקנה

### 1. דרישות מוקדמות
* **Python 3.9+** מותקן בשני המחשבים.
* חיבור רשת מקומית (LAN) המאפשר תקשורת בין הלקוח לשרת.
* פורט **5500** פתוח ב-Firewall של השרת.

### 2. התקנת ספריות
הרץ את הפקודה הבאה בטרמינל/CMD:
pip install -r requirements.txt

### 3. הגדרות משתני סביבה
צור קובץ בשם .env בתיקיית הפרויקט והזן את המפתחות שלך:

VT_API_KEY=your_virustotal_api_key_here

GENAI_API_KEY=your_gemini_api_key_here
### 4. הגדרת IP השרת
בתוך הקובץ CLIENT.py, עדכן את המשתנה self.server_ip לכתובת ה-IP האמיתית של מחשב השרת.

### 🚀 הוראות הפעלה
אתחול (פעם אחת): הרץ python CREATE_DB.py ליצירת מסד הנתונים.

הפעלת השרת: במחשב השרת, הרץ python SERVER.py.

הפעלת הלקוח: במחשב הלקוח, הרץ python CLIENT.py.
